## 🔧 Technical Details

### How It Works:
- Login credentials are **hardcoded in the frontend**
- No backend API calls for these test accounts
- Data stored in browser's **localStorage**
- JWT token is mocked for authentication
- Logout clears localStorage

### For Backend Integration:
- Once backend is ready, these default accounts will be replaced
- Real authentication will happen via API
- Database will store actual user data
- JWT tokens will be generated by backend

## ❓ Troubleshooting

### "Invalid credentials" error:
- Make sure you typed exactly: `admin` or `user` (lowercase)
- Password: `admin@123` or `user@123` (case-sensitive)
- No extra spaces

### "Still showing login page":
- Check if you clicked "Login" button
- Clear browser cache and try again
- Check browser console for errors (F12)

### "Can't see Admin Panel":
- Only available when logged in as `admin`
- User account cannot access Admin Panel
- Check sidebar menu items

### "Data not saving":
- Expected behavior - no backend yet
- Data is in browser localStorage temporarily
- Will persist until you clear cache or logout
